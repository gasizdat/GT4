I have a .NET MAUI application for building and viewing family trees (genealogy trees). I want you to redesign the UI so it becomes modern, elegant, intuitive, practical, and visually polished, while remaining realistic to implement in .NET MAUI using XAML.
Your task is not just to make the UI “look better”, but to improve the overall user experience and usability.
Please follow these requirements carefully:
1.	ROLE AND QUALITY LEVEL
•	Think and work like a professional product designer with strong experience in:
•	mobile and desktop UI/UX
•	information hierarchy
•	accessibility
•	form and data-entry design
•	tree / graph / relationship visualization
•	.NET MAUI XAML implementation
•	Provide design decisions that are practical and implementable, not abstract or purely conceptual.
2.	MAIN GOALS
•	Transform the UI from technical-looking to professional and user-friendly.
•	Make the interface cleaner, calmer, and more visually consistent.
•	Improve clarity, readability, navigation, and discoverability.
•	Reduce visual noise and unnecessary complexity.
•	Preserve the existing functionality unless there is a strong UX reason to reorganize it.
•	Keep the interface suitable for a genealogy/family-tree product: trustworthy, clear, structured, and emotionally warm, but not overly decorative.
3.	DESIGN PRINCIPLES Use these principles consistently:
•	clear visual hierarchy
•	consistent spacing and alignment
•	modern typography
•	restrained color palette
•	intuitive grouping of controls
•	minimal clutter
•	good empty-state and loading-state design
•	accessible contrast
•	comfortable touch targets
•	responsive/adaptive layout
•	reusable styles and components
•	platform-appropriate behavior for MAUI

•	Identify usability issues first.
•	Explain what is currently weak in the layout or interaction.
•	Propose specific UX improvements.
•	Then provide improved XAML.
•	If needed, reorganize sections, rename labels, improve button placement, simplify forms, improve card layouts, and refine navigation patterns.
•	Prefer solutions that reduce cognitive load.
•	Improve scanning and readability of dense data.
•	If a screen contains many controls, create a clearer structure using sections, cards, tabs, accordions, groups, or progressive disclosure where appropriate.

•	modern
•	clean
•	elegant
•	calm
•	trustworthy
•	polished
•	practical
Avoid:
•	overly bright colors
•	excessive borders
•	outdated gradients
•	cramped layouts
•	developer-style alignment
•	“raw default controls everywhere” appearance

•	Suggest a coherent design system for colors, spacing, corner radius, shadows, typography, and component states.
•	Use a restrained palette appropriate for genealogy/history/family context.
•	Make important actions visible, but avoid aggressive color usage.
•	Prefer subtle emphasis over flashy decoration.
7.	ACCESSIBILITY
•	Ensure good contrast and readability.
•	Use appropriate font sizes.
•	Avoid relying only on color to communicate meaning.
•	Make interactive elements easy to identify.
•	Keep layouts comfortable for both mouse and touch use.

•	All proposed solutions must be realistic for .NET MAUI XAML.
•	Prefer clean, maintainable XAML.
•	Reuse styles, resources, and templates where possible.
•	If useful, introduce:
•	ResourceDictionary
•	Styles
•	ControlTemplate
•	DataTemplate
•	VisualStateManager
•	reusable ContentView components

•	If the original structure is poor, refactor it into a cleaner XAML structure.
•	Do not break bindings unless absolutely necessary; preserve MVVM compatibility.
